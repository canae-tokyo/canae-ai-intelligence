# v1.1.1 ベンチマーク・評価分離

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 目的

v1.1.1では、公開ベンチマークとCANAE実務評価を独立データとして管理します。

ただしv1.1.xのスキーマ互換性ルールを優先し、`tools.json`の既存フィールド削除、名称変更、型変更、強制移行は行いません。

## 対象

| 対象 | 内容 |
| --- | --- |
| `data/benchmarks.json` | 公開ベンチマーク結果を結果単位で管理 |
| `data/canae-evaluations.json` | CANAE実務評価と根拠を評価単位で管理 |
| 参照ロジック | 新ファイルを優先し、旧`tools.json`フィールドへフォールバック |
| データ検証 | ID、toolId、日付、重複、混在禁止を検証 |
| UI | 既存表示を維持しつつ、分離データの参照元を表示 |

## 非対象

- `tools.json`既存フィールドの削除
- フィールド名称変更
- 型変更
- 既存データの強制移行
- UIの全面置換
- 旧フィールド廃止
- `noindex`解除
- 外部API、DB、ログイン機能

## benchmarks.json

`benchmarks.json`はツール単位ではなく、ベンチマーク結果単位で管理します。

| フィールド | 説明 |
| --- | --- |
| `id` | ベンチマーク結果ID |
| `toolId` | `tools.json`の実在ID |
| `benchmarkName` | SWE-bench Verifiedなどの名称 |
| `benchmarkVersion` | 評価バージョンまたは公開時点 |
| `score` | スコア。未確定時は`null` |
| `scoreUnit` | `percent` / `score` / `rank` / `points` |
| `rank` | 同一ベンチマーク内順位。未確定時は`null` |
| `scope` | 何を測る評価か |
| `sourceUrl` | 公式または信頼できる出典URL |
| `sourceType` | `official` / `third-party` / `legacy` |
| `verifiedAt` | CANAE確認日 |
| `comparability` | 比較可能性メモ |
| `dataStatus` | `draft` / `verified` / `archived` |
| `dataQuality` | `sample` / `partial` / `verified` |
| `notes` | 補足 |
| `changeLog` | 更新履歴 |

## canae-evaluations.json

`canae-evaluations.json`はCANAE独自評価と判断根拠を管理します。

| フィールド | 説明 |
| --- | --- |
| `id` | 評価ID |
| `toolId` | `tools.json`の実在ID |
| `evaluationVersion` | 評価基準バージョン |
| `overallGrade` | `S` / `A` / `B` / `C` |
| `scores` | 品質、速度、操作性、費用対効果、事業適合度 |
| `useCase` | 評価対象ユースケース |
| `evidence` | 評価根拠 |
| `evaluatedAt` | 評価日 |
| `evaluatedBy` | 評価主体。原則`CANAE` |
| `reviewStatus` | `draft` / `review` / `approved` / `archived` |
| `notes` | 補足 |
| `changeLog` | 更新履歴 |

## 段階移行ルール

UI参照は以下の優先順位で扱います。

1. `benchmarks.json`または`canae-evaluations.json`に有効データがあれば使用する。
2. 存在しない場合は`tools.json`既存フィールドへフォールバックする。
3. 両方ない場合は未設定表示にする。
4. 新旧値が競合する場合は、新ファイルを正本候補とし、PRレビューで確認する。

v1.1.1では旧フィールドを削除しません。完全分離や旧フィールド整理はv1.2以降で判断します。

## 検証ルール

`npm run validate:data`で以下を検証します。

- `toolId`が`tools.json`の実在IDと一致する。
- ベンチマークIDと評価IDに重複がない。
- 同一ツール、同一ベンチマーク、同一バージョンの重複登録がない。
- `verifiedAt`と`evaluatedAt`が`YYYY-MM-DD`形式である。
- `sourceUrl`はHTTPS URLである。
- 比較可能性メモが登録されている。
- CANAE評価を公開ベンチマークとして登録しない。
- CANAE評価スコアは1〜5の整数である。
