# データモデル設計

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 対象ファイル

- `news.json`: ニュース、重要度、検証状態
- `tools.json`: AIツール、ランキング、評価
- `models.json`: 将来のモデル単位データ
- `companies.json`: 企業、系列、製品階層
- `benchmarks.json`: 将来の公開ベンチマーク出典
- `categories.json`: 将来のカテゴリ定義

## news.json

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | string | ニュースID |
| `title` | string | タイトル |
| `company` | string | 関連企業 |
| `category` | GenreId | ジャンル |
| `importance` | high / medium / low | 重要度 |
| `publishedAt` | YYYY-MM-DD | 公開日 |
| `summary` | string | 要約 |
| `impact` | string | CANAEへの影響 |
| `sourceType` | official / media / sns | 出典種別 |
| `sourceUrl` | string | 出典URL |
| `status` | draft / verified / archived | 検証状態 |
| `sourceCheckedAt` | YYYY-MM-DD | 出典確認日 |

## tools.json

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | string | ツールID |
| `name` | string | ツール名 |
| `company` | string | 提供企業 |
| `category` | GenreId | ジャンル |
| `scores` | object | 6軸スコア |
| `benchmarkRank` | number/null | 公開Bmk順位 |
| `benchmarkSource` | string | 公開Bmk出典またはサンプル表記 |
| `benchmarkCheckedAt` | YYYY-MM-DD | 公開Bmk確認日 |
| `internalGrade` | S/A/B/C | CANAE実務評価 |
| `price` | string | 価格 |
| `apiAvailable` | boolean | API有無 |
| `commercialUse` | 可/要確認/不可 | 商用利用 |
| `lastUpdated` | YYYY-MM-DD | 最終更新日 |
| `officialUrl` | string | 公式URL |
| `tags` | string[] | タグ |

## models.json

モデル名、企業ID、系列、モダリティ、コンテキスト長、API有無、価格、強み、制限を管理する。

## companies.json

企業ID、企業名、国、公式URL、製品系列、提携・買収・出資などの関係を管理する。

## benchmarks.json

出典名、URL、取得日、対象ID、評価軸、スコア、補足を管理する。

## categories.json

カテゴリID、表示名、説明、親カテゴリ、表示順を管理する。
