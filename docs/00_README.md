# CANAE AI Intelligence Docs

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 運用ルール

- 仕様変更は先に該当ドキュメントへ反映し、必要に応じて実装へ反映する。
- Rule Aに該当する変更は、実装前に関係者合意を必須とする。
- `data/*.json`のサンプル値は実数ではない。公開・営業利用前にVerified値へ置き換える。
- 公開ベンチマークとCANAE評価は、UI、データ、文書上で必ず分離して扱う。
- 自動収集は候補作成までに限定し、`verified`昇格とProduction反映は人手確認を必須とする。
- 現行ステータスは`10_現行ステータス.md`を正とする。
- 本番運用開始時の確認結果は`09_本番運用開始報告書.md`を正とする。
- 未実装・今後対応予定の項目は`13_既知課題.md`へ記録する。
