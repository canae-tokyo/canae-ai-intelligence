# 運用ルール

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## ニュース登録ルール

ニュースは`Draft`、`Verified`、`Archived`で管理する。トップとジャンルページへ出す情報はVerifiedに限定する。

## Verified基準

一次情報または信頼できる情報源が確認済みで、企業名、日付、対象ツール、影響範囲が明確であること。SNS情報は単独でVerifiedにしない。

## ベンチマーク更新ルール

公開ベンチマークは出典名、取得日、対象モデル、評価軸を記録する。CANAE評価は検証者、検証日、検証タスク、評価理由を記録する。

## JSON更新方法

1. `data/*.json`を編集する。
2. JSON構文を確認する。
3. `npm run build`を実行する。
4. 表示、ランキング、相関図を確認する。
5. GitHubへcommitする。
6. Cloudflare build成功を確認する。

## 更新頻度

重要ニュースは随時、通常ニュースは週1回、CANAE評価と公開ベンチマークは月1回を目安とする。

## 情報源の優先順位

1. 公式発表、公式ドキュメント、公式価格表
2. 公開ベンチマークサイト
3. CANAE社内検証
4. 技術ブログ、専門メディア
5. SNS、コミュニティ投稿

## 差し戻しルール

出典なし、評価混在、評価理由なし、サンプル値と実測値の混同、ブランド表記違反、旧情報の更新扱いは差し戻す。
