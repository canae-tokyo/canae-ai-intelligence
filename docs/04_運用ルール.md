# 運用ルール

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## ニュース登録ルール

ニュースは`Draft`、`Verified`、`Archived`の3状態で管理します。主要画面に表示してよいのはVerifiedのみです。

| 状態 | 意味 | 表示可否 |
| --- | --- | --- |
| Draft | 未確認、調査中 | 原則非表示 |
| Verified | 出典確認済み、社内利用可 | 表示可 |
| Archived | 古い、無効、参照のみ | 通常非表示 |

## ニュース公開フロー

ニュースは以下の流れで扱います。

```text
draft
↓
review
↓
verified
↓
archived
```

| 工程 | 内容 | 実施者 |
| --- | --- | --- |
| draft | 候補ニュースを登録し、未確認として扱う | 更新担当 |
| review | 出典、日付、企業名、重要度、タグを確認する | レビュー担当 |
| verified | 表示可能な正式データとして扱う | 承認者 |
| archived | 古くなった情報、無効化された情報を退避する | 更新担当 |

## Verified基準

Verifiedにできる条件は以下です。

- 一次情報または信頼できる出典が確認されている。
- 日付、サービス名、企業名に誤りがない。
- 公開ベンチマークとCANAE評価のどちらに関係する情報か区別されている。
- サンプル値、推測、未確認情報である場合はその旨が明記されている。
- 社内提案に使っても誤解を招かない粒度で要約されている。

## ベンチマーク更新ルール

公開ベンチマークは出典名、取得日、対象モデル、評価軸を記録します。CANAE評価は検証者、検証日、検証タスク、評価理由を記録します。

公開ベンチマークの更新だけでCANAE評価を自動変更してはいけません。CANAE評価は実務検証を経て更新します。

## JSON更新方法

1. 対象JSONを編集する。
2. JSON構文を確認する。
3. サンプル値、実データ、未検証値の表示区分を確認する。
4. `npm run lint`を実行する。
5. `npm run build`を実行する。
6. 表示崩れ、ランキング順、Verified表示制御を確認する。
7. GitHubへcommitする。
8. Pull Requestを作成する。
9. CI成功を確認する。
10. Production反映後に主要画面を確認する。

## Production反映ルール

本番反映前に以下を確認します。

| 項目 | 条件 |
| --- | --- |
| CI | success |
| Lint | pass |
| Build | pass |
| 主要画面 | `/`, `/news`, `/genre/coding`を確認 |
| スマホ表示 | 360 / 375 / 390 / 430pxのいずれかで横スクロールなし |
| 404 | `404`応答 |
| noindex | HTML `noindex`と`X-Robots-Tag`を確認 |
| データ | Draft / Archivedが主要画面に出ていない |

Production反映後は、`10_現行ステータス.md`、`12_リリース一覧.md`、`changelog/CHANGELOG.md`の更新要否を確認します。

## Phase Freezeルール

Phase 1 / v1.0.1相当は凍結版として扱います。

| 項目 | ルール |
| --- | --- |
| 対象 | Phase 1 / v1.0.1相当 |
| 状態 | Freeze |
| 許可 | 重大不具合、セキュリティ、誤記、運用上必要な最小修正 |
| 禁止 | 新機能追加、大幅なUI変更、データ構造変更、`noindex`解除 |
| 新規開発 | v1.1ブランチまたはv1.1対象PRで実施 |
| 文書 | v1.0.1時点の正本文書は保持し、変更履歴で差分管理する |

凍結版へ変更が必要な場合は、以下を満たす必要があります。

1. 変更理由が不具合修正または運用維持である。
2. 影響範囲が限定されている。
3. `CHANGELOG`へ記録する。
4. 必要に応じて`10_現行ステータス.md`を更新する。
5. v1.1以降の仕様変更と混同しない。

## 更新頻度

| 対象 | 推奨頻度 |
| --- | --- |
| 重要ニュース | 随時 |
| 通常ニュース | 週1回 |
| CANAE評価 | 月1回または主要アップデート後 |
| 公開ベンチマーク | 月1回 |
| ジャンル・タグ体系 | 四半期ごと |

## 情報源の優先順位

1. 公式発表、公式ドキュメント、公式価格表
2. 公開ベンチマークサイト
3. CANAE社内検証
4. 技術ブログ、専門メディア
5. SNS、コミュニティ投稿

SNS情報は単独でVerifiedにしません。一次情報または社内検証と照合します。

## 差し戻しルール

以下に該当する登録・更新は差し戻します。

- 出典がない。
- 公開評価とCANAE評価が混在している。
- 評価理由がない。
- 実数とサンプル値が区別されていない。
- ブランド表記がガイドラインと異なる。
- 旧情報を更新扱いにしている。
