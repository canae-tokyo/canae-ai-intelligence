# ブランドガイドライン

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 正式名称

- 正式名称: `CANAE AI Intelligence`
- 短縮表記: `AI Intelligence`
- 会社表記: `株式会社CANAE`または`CANAE Inc.`
- 関連サービス: `Web Assist`

## ロゴ使用

v1.0ではテキストロゴを使用する。専用ロゴを追加する場合はCANAE、Web Assist、AI Intelligenceの階層が誤解されない表現にする。

## Web Assistとの関係

AI IntelligenceはWeb Assistの研究・検証部門として位置付ける。ただしWeb Assist本体の一部機能ではなく、独立した知識基盤として管理する。

## 表記ルール

推奨表記は`Powered by Web Assist`、`A service by CANAE Inc.`。公開ベンチマークとCANAE評価は別語として扱う。

## フッター

社内限定時は`CANAE AI Intelligence / Powered by Web Assist / Internal Use Only`を基準にする。

## ヘッダー

主表示は`CANAE AI Intelligence`、補助表示は`Powered by Web Assist`。

## コピー

推奨コピーは「AI業界を構造化・可視化するCANAEの知識基盤」「公開ベンチマークと実務評価を分離してAIを比較」。

## CTAルール

社内限定時はCTAを置かず、閲覧・確認・更新導線を優先する。公開PhaseではWeb Assist相談導線を設置できる。
