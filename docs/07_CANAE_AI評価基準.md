# CANAE AI評価基準

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 評価の基本方針

公開評価は参考情報、CANAE評価は実務判断のための内部評価として扱う。評価の中心はWeb制作、DX、業務効率化、コンテンツ制作、開発支援での有用性である。

## 公開評価

Artificial Analysis、Arena、LiveBench、公式ベンチマーク、第三者レビューを参照する。引用時は出典、取得日、対象モデル、評価軸を記録する。

## CANAE評価

| 評価 | 意味 | 利用判断 |
| --- | --- | --- |
| S | 積極採用 | 提案・制作・運用の中核候補 |
| A | 採用可 | 条件が合えば実務投入可能 |
| B | 限定採用 | 用途を絞れば利用可能 |
| C | 要注意 | 検証継続または非推奨 |

## 評価軸

- Web制作
- DX
- 業務効率
- 文章生成
- 画像生成
- 動画生成
- API
- 価格
- 学習コスト

## 数値スコア

90-100をS、80-89をA、70-79をB、0-69をCの目安とする。ただし最終評価は実務検証理由を必須とする。

## 評価プロセス

1. 評価対象を決める。
2. 公開情報を確認する。
3. CANAE実務タスクで検証する。
4. 軸別に採点する。
5. S/A/B/Cへ変換する。
6. 評価理由を記録する。
7. Verified化して反映する。

## 禁止事項

公開ベンチマークだけでCANAE評価を決めない。価格、UI、利用制限を無視しない。サンプル値を実測値として扱わない。
