# v1.1 データモデル確定

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 位置付け

この文書はv1.1の最初の正本です。実データ投入前に、`tools.json`と`news.json`のスキーマ、ID採番、出典管理、確認日、更新履歴の扱いを固定します。

v1.0.1相当はFreeze済みのため、この文書はPhase 1の上書きではなく、v1.1開発の入力仕様として扱います。

## v1.1の対象範囲

| 対象 | 方針 |
| --- | --- |
| `tools.json` | 実データ化の主対象 |
| `news.json` | 公式情報中心の実データ化対象 |
| `models.json` | v1.1では任意。必要に応じて設計のみ |
| `companies.json` | v1.1では任意。相関図精度向上時に検討 |

## 共通ルール

| 項目 | ルール |
| --- | --- |
| ID | 英小文字、数字、ハイフンのみ |
| 日付 | `YYYY-MM-DD` |
| URL | 公式URLまたは出典URLを優先 |
| ステータス | `draft` / `verified` / `archived` |
| 確認日 | `verifiedAt`で管理 |
| 確認者 | `verifiedBy`で管理 |
| 更新履歴 | `changeLog`配列で管理 |
| サンプル値 | `dataQuality: "sample"`で明示 |
| 実データ | `dataQuality: "verified"`で明示 |

## スキーマ互換性ルール

v1.1.xでは、実データ投入後の修正コストを抑えるため、後方互換性を維持します。

| 項目 | ルール |
| --- | --- |
| 既存フィールド削除 | 禁止 |
| 既存フィールド名変更 | 禁止 |
| 既存フィールドの型変更 | 原則禁止 |
| 新規フィールド追加 | 任意フィールドとして追加可能 |
| 必須フィールド追加 | v1.1.xでは原則禁止。必要な場合はv1.2以降で扱う |
| Enum値追加 | 後方互換性を確認した上で追加可能 |
| Enum値削除・名称変更 | 禁止 |
| スコア軸変更 | v1.2以降で扱う |
| JSON分割 | v1.2以降で扱う |

スキーマ変更を伴う提案は、v1.1.xでは原則として実装せず、`11_v1.1_開発ロードマップ.md`または次期ロードマップへ記録します。

例外として、重大不具合やデータ破損を防ぐために必要な変更は、影響範囲、移行手順、表示影響、CI結果を明記した上でレビューします。

## ID採番ルール

### tools.json

ツールIDはサービス名をベースにします。

```text
chatgpt
claude
gemini
github-copilot
canva-magic-studio
```

同一ブランドで複数製品がある場合は、製品名または用途を後ろに付けます。

```text
copilot-m365
copilot-github
gemini-app
gemini-api
```

### news.json

ニュースIDは日付と連番を組み合わせます。

```text
news-2026-07-21-001
news-2026-07-21-002
```

旧v1.0の`n001`形式はv1.1移行時に置換します。

## tools.json v1.1 Schema

### 必須フィールド

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | string | ツールID |
| `name` | string | 表示名 |
| `company` | string | 提供企業 |
| `family` | string | 製品系列 |
| `genre` | Genre | 主要ジャンル |
| `tags` | string[] | 検索・絞り込み用タグ |
| `description` | string | 80〜160字程度の概要 |
| `status` | `new` / `updated` / `stable` | UI表示用の更新状態 |
| `dataStatus` | `draft` / `verified` / `archived` | データ検証状態 |
| `dataQuality` | `sample` / `verified` / `partial` | データ品質 |
| `sourceUrl` | string | 公式または主要出典URL |
| `verifiedAt` | string | 確認日 |
| `benchmarkScore` | ScoreSet | 公開ベンチマーク由来の評価 |
| `internalScore` | ScoreSet | CANAE実務評価 |

### 任意フィールド

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `pricing` | string | 料金概要 |
| `pricingUrl` | string | 公式価格ページ |
| `apiAvailable` | boolean | API提供有無 |
| `apiUrl` | string | APIドキュメントURL |
| `commercialUse` | `yes` / `no` / `conditional` / `unknown` | 商用利用可否 |
| `japaneseSupport` | `native` / `supported` / `limited` / `unknown` | 日本語対応 |
| `primaryUseCases` | string[] | 主用途 |
| `limitations` | string[] | 注意点 |
| `benchmarkSource` | BenchmarkSource | 公開ベンチマーク出典 |
| `canaeEvaluation` | CanaeEvaluation | CANAE評価根拠 |
| `verifiedBy` | string | 確認者または責任者 |
| `changeLog` | DataChange[] | データ更新履歴 |

### ScoreSet

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `overall` | number | 総合 |
| `performance` | number | 性能 |
| `japanese` | number | 日本語 |
| `usability` | number | 操作性 |
| `speed` | number | 速度 |
| `automation` | number | 自動化 |

スコアは0〜100の整数とします。未検証の場合でも空欄にはせず、`dataQuality`と表示文言で未検証を明示します。

## BenchmarkSource

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `name` | string | yes | 出典名 |
| `url` | string | yes | 出典URL |
| `retrievedAt` | string | yes | 取得日 |
| `method` | string | no | 算定方法の補足 |
| `note` | string | no | 注意点 |

公開ベンチマークはCANAE評価に自動反映しません。公開ベンチマークは外部評価、CANAE評価は実務検証として独立管理します。

## v1.1移行手順

1. 既存`tools.json`と`news.json`の棚卸しを行う。
2. v1.1必須フィールドを追加する。
3. `dataQuality`を`sample`、`partial`、`verified`に分類する。
4. 公式出典URLと確認日を登録する。
5. `draft` / `verified` / `archived`の表示制御を確認する。
6. `npm run lint`と`npm run build`を実行する。
7. 主要画面でサンプル値と実データが誤認されないことを確認する。
