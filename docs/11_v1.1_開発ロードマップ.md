# v1.1 開発ロードマップ

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## 対象

v1.1は、UIや本番基盤の大幅変更ではなく、データ品質と運用強化を対象にします。

Phase 1 / v1.0.1相当は凍結版として保持し、v1.1の新規開発はv1.1対象ブランチまたはv1.1対象PRで管理します。

## 目的

- サンプル値を実データへ置換する。
- 公開ベンチマークの出典と確認日を明確にする。
- CANAE実務評価の判断根拠を運用可能な形にする。
- ニュース、ツール、モデル、会社情報を継続更新できる状態にする。
- 自動収集・更新補助機能の設計を開始する。

## 優先順位

| 優先 | 項目 | 内容 |
| --- | --- | --- |
| P0 | 実データ置換 | `tools.json` / `news.json`のサンプル値を実データへ置換 |
| P0 | 出典整備 | 公開ベンチマークの出典、確認日、対象モデルを明記 |
| P1 | AIツールDB拡充 | 新規ツール追加、カテゴリ整理、価格・API情報更新 |
| P1 | CANAE評価運用 | 評価基準、検証者、検証日、承認フローを整備 |
| P2 | 自動収集設計 | RSS、公式ブログ、GitHub Releases、リリースノート収集の要件定義 |
| P2 | 更新補助 | JSON更新チェック、差分確認、レビュー補助の検討 |

## 除外事項

v1.1では以下を原則対象外とします。

- ログイン機能
- 外部DB導入
- 外部API連携の本番実装
- 課金・会員機能
- 大規模UIリニューアル
- SEOインデックス公開
- `noindex`解除
- Phase 1正本文書の上書き

## 完了条件

| 項目 | 条件 |
| --- | --- |
| データ | 主要ツールとニュースのサンプル値が実データへ置換されている |
| 出典 | 公開ベンチマークに出典名、URL、確認日がある |
| 評価 | CANAE評価に評価理由、検証日、検証者または責任者がある |
| 表示 | サンプル値が実データのように誤認されない |
| CI | `npm ci`、`npm run lint`、`npm run build`が通る |
| 本番 | Cloudflare Production反映後に主要画面を確認済み |
| 文書 | `CHANGELOG`と現行ステータスが更新済み |

## 推奨作業順

1. v1.1データモデルの確定
2. 現行JSONの棚卸し
3. 実データ化対象フィールドの確定
4. 公開ベンチマーク出典ルールの適用
5. `tools.json`更新
6. `news.json`更新
7. CANAE評価サンプルの見直し
8. 表示確認
9. CI / build確認
10. PR作成
11. Production反映

v1.1データモデルは`14_v1.1_データモデル確定.md`を正とします。

v1.1.xではスキーマ後方互換性を維持します。既存フィールドの削除、名称変更、型変更、必須フィールド追加は原則禁止し、破壊的なスキーマ変更はv1.2以降で扱います。
