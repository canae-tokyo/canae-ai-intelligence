# AI Intelligence プロジェクト概要 v1.0

## Design Principles

1. ブランド・URL・GitHub・インフラは独立した4レイヤーとして管理する。
2. AI IntelligenceはWeb Assistの研究・検証部門として位置付けるが、システムは完全に独立させる。
3. 公開ベンチマークとCANAE独自評価を混同しない。
4. AIニュースサイトではなく、AI業界を構造化・可視化する知識基盤を目指す。
5. ニュースよりもCANAEの実務検証を価値の中心に据える。

## プロジェクト概要

CANAE AI Intelligenceは、AIツール、モデル、企業系列、ニュース、評価を構造化して可視化する社内知識基盤である。

## 開発背景

AIツールの増加により、単なるニュース収集では実務判断に不足する。CANAEではWeb制作、DX、業務効率化で使えるAIを継続評価する必要がある。

## Web Assistとの関係

AI IntelligenceはWeb Assistの研究・検証部門として位置付ける。ただしURL、GitHub、Cloudflare、運用は独立管理とする。

## ブランド戦略

主ブランドはCANAE、実務導線はWeb Assist、評価基盤はAI Intelligenceとして階層化する。

## 設計思想（4レイヤー分離）

- ブランド: CANAE / Web Assist / AI Intelligence
- URL: `ai.canae.tokyo`
- GitHub: `canae-tokyo/canae-ai-intelligence`
- インフラ: Cloudflare Workers

## サービスコンセプト

AI業界を構造化・可視化し、公開ベンチマークとCANAE実務評価を分離して比較できる社内ダッシュボード。

## 差別化戦略

ニュース量ではなく、CANAEの実務検証、評価理由、採用判断への接続を価値の中心に置く。

## ターゲット

経営者、Web Assist担当者、制作・開発担当者、AI導入提案担当者。

## ロードマップ

- Phase 1: 静的ダッシュボード、JSON管理、noindex運用
- Phase 2: Cloudflare Access、独自ドメイン接続
- Phase 3: 評価軸と相関図の拡充
- Phase 4: データ更新フローの半自動化
- Phase 5: 公開レポート、問い合わせ導線、外部発信

## 将来構想

AI評価レポート、Web Assist提案連携、社内ナレッジ検索、モデル別比較、業種別推奨AIリストへ拡張する。

## 成功指標（KPI）

- Verifiedニュース登録数
- CANAE評価済みツール数
- 提案資料への引用数
- 月次更新継続率
- Web Assist案件での活用数
